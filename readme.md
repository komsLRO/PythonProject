# Тема:

Разработать Telegram‑бота‑планировщика дня, который помогает пользователю создавать расписание, получать напоминания о делах и отслеживать выполнение задач в одном мессенджере.

# Цель проекта:

Создать удобного Telegram‑бота‑планировщика, который позволяет пользователю быстро заносить дела на день, получать напоминания в нужное время и видеть, что уже выполнено, без установки отдельных приложений.

# Актуальность:

Люди часто забывают о задачах, встречах и дедлайнах, особенно когда дела записаны в разных местах: заметки, календарь, мессенджеры. Бот‑планировщик собирает всё в одном чате, которым человек и так пользуется каждый день.

Telegram‑бот не требует установки нового приложения, работает на любом устройстве и может отправлять напоминания 24/7, что делает его удобным инструментом для студентов и занятых людей.

# Задачи проекта:

1. Спроектировать структуру бота‑планировщика: команды для работы с задачами, слой работы с базой данных и модуль напоминаний.
2. Реализовать хранение пользователей и задач (текст задачи, дата, время, статус «выполнено/не выполнено») в базе данных.
3. Сделать команды и кнопки для добавления, просмотра, редактирования и удаления задач на день/неделю.
4. Реализовать планировщик, который регулярно проверяет задачи и отправляет напоминания в нужное время с учётом часового пояса пользователя.
5. Добавить простой интерфейс настроек (например, включение/выключение напоминаний утром/вечером, тихие часы).
6. Провести тестирование бота на нескольких пользователях и доработать сценарии и тексты сообщений.

# Библиотеки:

1. Python – основной язык разработки.
2. aiogram 3.x – работа с Telegram Bot API, обработка команд и сообщений.
3. sqlite3 – встроенная база данных для хранения пользователей и задач.
4. asyncio – работа с асинхронными задачами и планировщиком напоминаний.
5. requests или httpx – при необходимости для внешних сервисов (например, синхронизация с календарём в будущем).

# Архитектура проекта:

* Модуль бота (aiogram)
* Обрабатывает команды и кнопки пользователя: добавление задачи, список задач на сегодня, отметка «сделано», настройки.
* Слой работы с БД (SQLite)
* Таблица пользователей (id, имя, часовой пояс, настройки уведомлений).
* Таблица задач (id задачи, id пользователя, текст, дата, время, статус, тип напоминания).
* Планировщик задач
* Фоновый модуль (на базе asyncio или встроенного scheduler в aiogram), который раз в N секунд/минут проверяет базу и отправляет напоминания по тем задачам, у которых подошло время.
* Модуль настроек и профиля
* Логика изменения часового пояса, включения/выключения напоминаний, выбора формата времени (24 часа/12 часов).
* При желании отдельно можно выделить слой «сервисов» — функции, которые не завязаны на Telegram напрямую (работа с задачами, фильтрация по дате), чтобы код было проще поддерживать.

# Основные возможности:

1. Добавление задач на сегодняшний день или конкретную дату с указанием времени.
2. Просмотр задач
3. список задач на сегодня;
4. задачи на выбранный день;
5. задачи на ближайшую неделю.
6. Отметка задач как «выполнено» и возможность скрывать выполненные задачи из основного списка.
7. Автоматические напоминания в указанное время (например, за 10 минут до задачи и в момент задачи).
8. Простые настройки профиля: часовой пояс, время начала и конца дня, тихие часы, формат времени.

# Дополнительные функции:

1. Шаблоны задач: возможность быстро добавить типичное дело по кнопке (например, «пары», «тренировка», «домашка по матану»).
2. Повторяющиеся задачи: ежедневные, по будням, по определённым дням недели.
3. Мини‑статистика: сколько задач запланировано и сколько выполнено за день или неделю (простые числа, без жёсткой аналитики).
4. Экспорт списка задач за день/неделю в текстовом виде.
